## Языки

Внизу таблицы 5 вкладок: `en`, `fr`, `de`, `nl`, `ar`.

**Каждая статья должна быть переведена на нужные языки** — добавляешь по одной строке в каждый лист с тем же `slug`. Если на каком-то языке статьи нет — сайт покажет английскую версию (fallback).

## Структура строки (колонки A → I)

| Колонка | Что это | Пример |
|---|---|---|
| **A — slug** | URL-часть статьи (только латиница + тире) | `belgian-team-wins-euro` |
| **B — title** | Заголовок | `Belgian team wins Euro 2026 qualifier` |
| **C — excerpt** | Превью (1-2 предложения) — на карточке | `Red Devils beat France 2-1 at home.` |
| **D — image_url** | Прямая ссылка на картинку (jpg/png) | `https://images.unsplash.com/photo-...` |
| **E — content** | Полный текст статьи (Markdown — см. ниже) | `## Recap\n\nThe **Red Devils**...` |
| **F — category** | Категория | `Football`, `Tennis`, `News`, `Promotion` |
| **G — author** | Автор | `Andrei G` |
| **H — date** | Дата публикации (YYYY-MM-DD) | `2026-05-12` |
| **I — featured** | `true` чтобы закрепить статью первой | `true` или пусто |

### Важно про slug
- **Только латиница, цифры, тире** — никаких пробелов, запятых, точек
- **Должен быть уникальным** в пределах одного языка
- **Не меняй slug опубликованной статьи** — иначе старая ссылка перестанет работать
- Примеры хороших: `nadal-comeback-rome`, `euro-2026-final`, `champions-league-draw`

### Картинки
- Лучше всего — **Unsplash** (бесплатно, https://unsplash.com). Найди фото → "Copy image address" → вставь URL
- Размер не критичен — мы автоматом обрезаем под нужный формат
- **Стандартный размер**: рекомендуется 1200×675 (16:9)
- Если у тебя свои фото — заливай в Google Drive, делай "anyone with link", вставляй прямую ссылку. Или используй https://imgur.com / https://cloudinary.com (бесплатно)

### Markdown в колонке E (content)

Текст статьи поддерживает базовое форматирование. Чтобы вставить перенос строки в ячейке Google Sheets — нажми **Ctrl + Enter** (Cmd + Enter на Mac).

```
## Заголовок секции

Обычный параграф текста. Можно сделать **жирным** или *курсивом*.
Ссылки делаются так: [текст ссылки](https://example.com).

### Заголовок поменьше

- Маркированный список
- Второй пункт
- Третий пункт

> Цитата на отдельной строке.
> Может быть на нескольких строках.

Ещё один параграф.
```

**Что поддерживается:**
- `## заголовок` (h2)
- `### заголовок` (h3)
- `**жирный**`
- `*курсив*`
- `[текст](url)` — ссылки
- `- пункт` — маркированный список
- `> цитата`
- Пустая строка между блоками = новый параграф

**Что НЕ работает** (пока): таблицы, картинки внутри текста, код-блоки. Если очень нужно — напиши в задачу.

## Categories

Используй одну из:
- `Football`, `Basketball`, `Tennis`, `Hockey`, `Esports`, `Volleyball`
- `News` — общая новость
- `Promotion` — промо-материал
- `Match Recap` — обзор матча
- `Preview` — анонс матча
- `Analysis` — аналитика
- `Transfer` — трансферные новости

## Featured

В колонке `featured` поставь `true` для **одной** статьи, которую хочешь видеть наверху страницы новостей. Если несколько с `true` — покажется первая по дате.

## Когда статья появится на сайте

- **На странице /news** — через 5 минут (мы кэшируем результат)
- **На главной странице** в блоке "Latest News" — те же 5 минут
- **Если очень срочно нужно** — Андрей может вручную сбросить кэш через Vercel панель

## Удалить статью

Удали строку из всех языковых листов. Через 5 минут пропадёт с сайта. Если кто-то уже сохранил ссылку — попадёт на страницу "Article not found".

## Что показывается на /news

Порядок:
1. **Featured статьи из Sheets** (если есть `true` в колонке I)
2. **Обычные статьи из Sheets** (свежие первыми)
3. **Внешние новости** с NewsData.io (автоматический парсинг спорт-СМИ)
4. **Промо-баннеры Vivat Sport** — вставляются 1 на каждые 3 статьи

## Если что-то сломалось

- 🟢 Sheets недоступны → сайт автоматически покажет только внешние новости + промо
- 🟢 Картинка не загружается → автоматически подставляется placeholder
- 🟡 Статью открыли по старому slug → страница "Article not found"
- 🔴 Если на сайте видишь ошибку — пиши Андрею с скриншотом
